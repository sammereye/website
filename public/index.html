DORISv2 (Dig our RIS)
2020 — Ongoing
Ruby on Rails
ElasticSearch
JavaScript
MSSQL Server
WHAT
DORISv2, which stands for Dig our RIS, is a Ruby on Rails web application that gives researchers and physicians the ability to query through patient exams. Users are able to search and filter based on their needs and then output the data to .csv format.
WHY
Since patient-study data comes from multiple sources, there is a need for a centralized application that allows users to search through the entire organization's studies. Due to technological and design advancements, the first version of DORIS was created roughly 10 years ago and was in need of an overhaul.
HOW
It all starts with the data inflow. The DORISv2 data comes from four independent sources. To streamline the process as much as possible, I setup an API that takes in HL7 messages, parses the data as needed and inserts it into each table. The report text of the study is then indexed through Elasticsearch and made available to be searched for on the front-end. The design of the application utilizes an IU framework called Rivet that is meant to normalize the design style across all Indiana University websites/applications. To handle the search functionality, jQuery Ajax is used to create POST API request to retrieve searches without the need to refresh the page.
WHO
DORISv2 has multiple use cases and users. The primary use case is to allow researchers to search for specific keywords and then filter the results down to a dataset they can use in their research. Another use case is for administration to understand the productivity levels of sections and their individual radiologists.
RECIST Calculator
2019 — 2021
Ruby on Rails
JavaScript
MSSQL Server
WHAT
The RECIST Calculator is a Ruby on Rails web application that allows radiologists and oncologists to add patients, add associated lesions/tumors and enter in measurements each time they visit for an appointment. The measurement data is then used to track how the patient is progressing throughout their treatment.
WHY
Prior to the RECIST Calculator, radiologists and oncologists would store individual patient visits in a shared drive. This led to major organization and patient visitation issues. A singular web application that stores all of the patient data in MSSQL database keeps everything simple and error-resistant.
HOW
The RECIST Calculator works by having radiologist and/or oncologists enter in patient data and then regularly update lesion/tumor measurements. It all runs on Ruby on Rails and the data is inserted and retrieved from a MSSQL server. Quite a bit of the heavy work is done on the front end such as creating charts, allowing printing and formatting data within structured tables. On the back-end, calculations are made to determine the trends of the treatment and to store relevant data. Another thing to highlight is the need to secure Protected Health Information (PHI). This is done by logging all interactions such as viewing a patient or adding notes.
WHO
The primary stakeholder of the RECIST Calculator are the Riley Children's Hospital radiologists and oncologists. They had a need for an updated electronic system that allowed to them to update lesions/tumors and monitor treatments for their patients.
Westriver High School LMS
2017 — 2018
NodeJS
Express
MySQL
Handlebars
WHAT
The Westriver High School Learning Management System (LMS) allows school administrators to create courses, multiple sections for each course, student accounts, teacher accounts, and departments. They can then link students to classes
WHY
Learning Management Systems (LMS) are an essential part of modern day schooling. They allow schools to keep classes and their associated assignments all online. This allows students to access all of their classes from anywhere and complete their assignments at anytime.
HOW
The Westriver High School LMS is created using NodeJS, Express, Handlebars and MySQL. NodeJS is the overlying runtime for processing JavaScript on the back-end. Express is then used as the framework for the site and helps facilitate the routing. Handlebars is the templating language that allows the application to dynamically show/modify data from the back-end to the front-end. Lastly, MySQL is the database that stores all of the data including login names and md5 hashed and salted passwords.
WHO
Although Westriver High School is a fictional school, every school nowadays needs to be digital and keep up with the rising needs of students and parents. This is the approach that I felt could be a solution to a school's need.
